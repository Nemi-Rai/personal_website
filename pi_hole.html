Author: Nemi Rai

Date: August 2021

> Summary

Pi-hole is a DNS server application that can be run on most *nix operating systems. Its primary purpose is to drop any requests to known ad-serving domains, however since the projects release many new features have been implemented.

> Recursive DNS Server

This implementation uses a low-cost Raspberry Pi microcomputer to use to create a recursive, and caching DNS resolver. Under the hood, the unbound service is used to add this extra functionality. A tutorial, as well as configuration files can be found in the doc.

A standard installation of pi-hole uses third-party upstream DNS servers to resolve requests. However if the recursive feature is installed, Pi-hole will use unbound to seek out the authoritative servers for a given request (if it is not already locally cached). The benefit of using a personal recursive DNS server is having extra privacy from third parties that you would otherwise share DNS requests with.

> Blocklists

Pi-hole comes with a standard list of ad-serving domains, this can be expanded upon by adding blocklist collections. These collections can be found by searching online, here's a few well known repositories:
